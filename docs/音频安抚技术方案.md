# 宠博士·云伴 音频安抚技术方案

> **版本：** v1.0 | **日期：** 2026-05-14 | **关联文档：** PRD v1.0 第6章

---

## 1. 方案概述

### 1.1 技术定位

音频安抚引擎是宠博士·云伴的**核心技术模块**，负责从环境音采集到安抚语音播放的完整闭环。与通用音频处理不同，本方案必须具备以下三个特性：

1. **实时性：** 从声音采集到触发判断必须在毫秒级完成，确保及时响应宠物叫声
2. **可靠性：** 在浏览器休眠、后台标签页等边缘场景下尽可能保持监听
3. **低误触发：** 通过多条件判定机制区分宠物叫声和环境噪音

### 1.2 方案范围边界

- **包含：** 音频采集、实时分贝分析、阈值触发引擎、安抚语音播放、录音管理
- **不包含：** AI 叫声分类识别（需训练模型）、实时视频、多声道空间音频、回音消除

## 2. 音频采集方案

### 2.1 API 选型

| 方案 | 优势 | 劣势 | 选择 |
|------|------|------|------|
| `getUserMedia` | 浏览器原生、零依赖、权限管理成熟 | 采样率不可精细控制 | ✅ 采用 |
| Web Audio API `AudioContext` | 实时处理、低延迟、节点图灵活 | 需要一定学习成本 | ✅ 采用 |
| AudioWorklet | 最低延迟、独立线程 | 兼容性有限（Safari 较新版本才支持） | ❌ MVP 不采用 |
| WebRTC | 支持远程传输 | 过重，不需要网络传输 | ❌ 不采用 |

### 2.2 音频管道搭建

```javascript
// 1. 请求麦克风权限
const stream = await navigator.mediaDevices.getUserMedia({ audio: true })

// 2. 创建 AudioContext（延迟启动，避免 Safari 自动播放限制）
const ctx = new (window.AudioContext || window.webkitAudioContext)()

// 3. 创建 AnalyserNode（分析器）
const analyser = ctx.createAnalyser()
analyser.fftSize = 256        // 128 个频点，平衡精度与性能
analyser.smoothingTimeConstant = 0.3  // 轻度平滑，保持响应速度

// 4. 连接管道
const source = ctx.createMediaStreamSource(stream)
source.connect(analyser)
// 注意：不 connect 到 destination，避免啸叫
```

### 2.3 FFT 参数选择

| 参数 | 可选值 | 本项目选择 | 理由 |
|------|--------|----------|------|
| `fftSize` | 32-32768 (2^n) | 256 | 128 频点，15ms 刷新，性能/精度平衡 |
| `smoothingTimeConstant` | 0-1 | 0.3 | 保留响应速度，轻度平滑 |
| 频点数量 | fftSize / 2 | 128 | 覆盖 0-22kHz 全频段 |

## 3. 分贝计算方案

### 3.1 算法选择

#### 方案 A：时域 RMS 法（采用）

```javascript
// 从 TimeDomainData 计算 RMS 分贝
function calculateDb(timeData) {
  let sum = 0
  for (let i = 0; i < timeData.length; i++) {
    const normalized = (timeData[i] - 128) / 128  // Uint8 → [-1, 1]
    sum += normalized * normalized
  }
  const rms = Math.sqrt(sum / timeData.length)
  const db = rms > 0 ? 20 * Math.log10(rms) + 94 : 0  // 94dB 为校准偏移
  return Math.round(db)
}
```

**优势：** 计算量小（O(n)，n=128），每帧刷新，实时性好
**局限：** 不区分频率，任何音量达到阈值都触发

#### 方案 B：频域加权法（预留给后续）

```
// 从 FrequencyData 提取目标频段能量
// 狗叫主频：600-2000Hz
// 猫叫主频：400-1200Hz
const freqData = new Uint8Array(analyser.frequencyBinCount)
analyser.getByteFrequencyData(freqData)
const targetEnergy = freqData.slice(targetRange).reduce((a, b) => a + b, 0)
// 判断目标频段能量是否超过阈值
```

### 3.2 分贝映射参考

| 场景 | 典型分贝 |
|------|---------|
| 安静房间 | 30-40 dB |
| 正常交谈 | 50-60 dB |
| 小型犬叫 | 60-75 dB |
| 中型犬叫 | 70-85 dB |
| 大型犬叫 | 80-95 dB |
| 城市街道 | 70-85 dB |

**本项目默认阈值：75dB**（覆盖大多数中小型犬叫，同时过滤正常环境噪音）

## 4. 触发引擎设计

### 4.1 状态机设计

```
┌─────────────────────────────────────────────────┐
│                   LISTENING                      │
│  每帧计算 dB，绘制波形                            │
│                                                  │
│  dB >= 75?                                        │
│    ├── NO → 重置 accumulator, 继续               │
│    └── YES → accumulator += deltaTime            │
│              accumulator >= 2000ms?              │
│                ├── NO → 继续累积                  │
│                └── YES → TRIGGER!                │
│                          │                       │
└──────────────────────────┼───────────────────────┘
                           ▼
┌─────────────────────────────────────────────────┐
│                   TRIGGERED                      │
│  emoji = 😱, banner = visible                    │
│  随机选一条 soothing_voice → 播放                 │
│  记录 anxiety_event 到 Supabase                   │
│  4 秒后 banner 消失                               │
│                          │                       │
└──────────────────────────┼───────────────────────┘
                           ▼
┌─────────────────────────────────────────────────┐
│                   COOLDOWN                       │
│  忽略所有触发条件                                 │
│  持续 60 秒（可配置 30-300s）                     │
│                          │                       │
│  冷却结束 → 回到 LISTENING                       │
└─────────────────────────────────────────────────┘
```

### 4.2 误触发防护机制

| 防护层 | 机制 | 参数 |
|--------|------|------|
| **第一层：持续确认** | 非瞬时触发，需持续超过阈值 N 秒 | 默认 2s |
| **第二层：冷却间隔** | 两次安抚之间强制间隔 | 默认 60s |
| **第三层：阈值余量** | 阈值设在高分贝区间（狗叫范围） | 默认 75dB |
| **第四层（预留）** | 频域过滤 | 600-2000Hz 狗叫主频 |

### 4.3 播放时的防啸叫

- 播放安抚语音时不触发新的检测（`isPlayingSoothing` 标志位）
- 播放前设置 `aboveThresholdSince = null`，重置累积计时器
- 冷却期间不响应任何触发

## 5. 安抚语音播放方案

### 5.1 语音存储架构

```
录音流程：
  getUserMedia → MediaRecorder → WebM Blob → Supabase Storage

播放流程：
  Supabase Storage URL → new Audio(url) → audio.play()
```

### 5.2 录音技术

```javascript
// 1. 获取录音权限
const stream = await navigator.mediaDevices.getUserMedia({ audio: true })

// 2. 创建 MediaRecorder
const recorder = new MediaRecorder(stream, { mimeType: 'audio/webm' })
recorder.ondataavailable = (e) => chunks.push(e.data)

// 3. 停止时上传到 Supabase
recorder.onstop = async () => {
  const blob = new Blob(chunks, { type: 'audio/webm' })
  const { data } = await supabase.storage
    .from('soothing-voices')
    .upload(`${Date.now()}.webm`, blob)
}
```

### 5.3 播放策略

| 策略 | 实现 | 适用场景 |
|------|------|---------|
| 随机播放 | `voices[Math.floor(Math.random() * voices.length)]` | 降低宠物脱敏风险 |
| 顺序播放 | 维护 index 指针，循环递增 | 用户明确偏好顺序 |
| 单段播放 | MVP 阶段退化策略 | 用户只录了一段 |

**本项目采用：随机播放（默认）+ 未来可选顺序播放**

### 5.4 播放降级策略

```
获取 soothing_voices？
  ├── 有语音 → 加载 Supabase URL → Audio.play()
  │              ├── 成功 → 正常播放
  │              └── 失败 → 播放振荡器提示音（正弦波 440→554→440Hz，1秒淡出）
  └── 无语音 → 显示提示文字 + 播放振荡器提示音
```

### 5.5 兼容性处理

| 浏览器 | AudioContext | getUserMedia | MediaRecorder | 状态 |
|--------|-------------|--------------|---------------|------|
| Chrome 80+ | ✅ | ✅ | ✅ | 完全支持 |
| Edge 80+ | ✅ | ✅ | ✅ | 完全支持 |
| Firefox 75+ | ✅ | ✅ | ✅ | 完全支持 |
| Safari 14+ | ✅ | ✅ | ✅ | 需要用户手势触发 |
| iOS Safari 14+ | ✅ | ✅ | ⚠️ 部分支持 | 建议 PWA 模式 |

## 6. 性能优化

### 6.1 帧率控制

- `requestAnimationFrame` 随显示器刷新率，通常 60fps
- 每帧计算量：128 次乘法 + 平方 + 求和 ≈ 0.01ms
- Canvas 波形绘制：128 次 `lineTo` 调用 ≈ 0.05ms
- 总单帧耗时 <0.1ms，对主线程影响可忽略

### 6.2 内存管理

- `timeData` / `frequencyData` 数组在 `startListening` 时分配，复用不重建
- 停止监听时释放 `MediaStream`、`AudioContext`、取消 `rAF`
- 波形 Canvas 不存储历史数据，每帧覆盖绘制

### 6.3 电池优化

- 使用 `requestAnimationFrame`（而非 `setInterval`），标签页后台时自动暂停
- 后续可考虑：检测到连续静音超过 5 分钟后降低检测频率
