# 宠博士·云伴 情绪评估体系设计

> **版本：** v1.0 | **日期：** 2026-05-14 | **关联文档：** PRD v1.0 第9章

---

## 1. 评估体系概述

### 1.1 为什么需要情绪评估

宠物情绪安抚与传统功能型 App 最大的区别在于：**"功能在运行" ≠ "安抚在生效"**。即使音频监听和语音播放功能正常运行，也无法直接回答以下核心问题：

- 宠物的分离焦虑到底有多严重？
- 主人录制的安抚语音是否真的有效？
- 宠物的情绪状态是在改善还是恶化？
- 什么时间段宠物最容易焦虑？

因此宠博士·云伴采用 **数据驱动的情绪评估体系**，将原始音频事件转化为可理解、可追踪的情绪指标。

### 1.2 评估设计原则

1. **对用户友好：** 用自然语言总结代替原始数据表格，让不懂技术的宠物主人也能看懂
2. **可量化：** 所有指标都有明确的数据来源和计算公式，不依赖主观判断
3. **有时效：** 支持日/月两级时间维度，揭示情绪变化趋势
4. **诚实：** 评估仅基于分贝数据，标注局限性（无法100%确定宠物真实情绪状态）

## 2. 核心评估指标

### 2.1 指标定义总表

| 指标ID | 指标名称 | 定义 | 计算公式 | MVP 目标 | 数据来源 | 计算频率 |
|--------|---------|------|---------|--------|---------|---------|
| E01 | 焦虑频次 | 单位时间内的焦虑触发次数 | anxietyCount / (durationMinutes / 60) | - | anxiety_events + guard_sessions | 每次守护结束 |
| E02 | 峰值强度 | 当日检测到的最高分贝值 | MAX(peak_db) | - | anxiety_events | 每次守护结束 |
| E03 | 焦虑总时长 | 当日处于焦虑状态的累计时长（分钟） | SUM(duration_sec) / 60 | - | anxiety_events | 每次守护结束 |
| E04 | 高发时段 | 焦虑事件最集中的时间段 | 按小时分组，COUNT 最大值对应的小时 | - | anxiety_events.triggered_at | 每次守护结束 |
| E05 | 安抚介入率 | 焦虑事件中实际播放了安抚语音的比例 | COUNT(voice_played_id IS NOT NULL) / COUNT(*) × 100% | >95% | anxiety_events | 每次守护结束 |
| E06 | 情绪环比 | 与昨日的焦虑总时长变化 | (昨日焦虑分钟 - 今日焦虑分钟) / 昨日焦虑分钟 × 100% | - | daily_reports.stats_json | 每日 |
| E07 | 月平静率 | 一个月中有守护数据的天数中，无焦虑天数的占比 | (平静天数 / 总有数据天数) × 100% | >80% | daily_reports 聚合 | 每月 |

### 2.2 指标详解

#### E01：焦虑频次

```
定义：平均每小时检测到的焦虑事件次数

为什么重要：
衡量宠物分离焦虑严重程度的核心指标。频次越高，说明宠物越不安。

计算示例：
守护 8 小时，检测到 6 次焦虑事件
E01 = 6 / 8 = 0.75 次/小时

解读：
<0.5 次/小时 → 轻度焦虑
0.5-2 次/小时 → 中度焦虑
>2 次/小时 → 重度焦虑
```

#### E02：峰值强度

```
定义：所有焦虑事件中，记录到的最高分贝值

为什么重要：
反映宠物情绪波动的最大烈度。一次剧烈的嚎叫（90dB+）
比多次轻微呜咽（60dB）更值得关注。

解读：
<80dB → 轻度吠叫
80-90dB → 中度吠叫/嚎叫
>90dB → 激烈嚎叫（可能伴随破坏行为）
```

#### E04：高发时段

```
定义：焦虑事件按小时聚合后，数量最多的小时区间

为什么重要：
帮助主人发现模式：
- 刚出门 1 小时内触发 → "分离即时反应"
- 下午 3 点触发 → 可能与外界噪音（快递、垃圾车）有关
- 傍晚触发 → 可能与饥饿/排泄需求有关

使用场景：
主人可以根据高发时段调整安抚策略
（如出门前增加运动量、在特定时段安排朋友上门）
```

#### E06：情绪环比

```
定义：与前一天相比，焦虑总时长的变化百分比

为什么重要：
这是衡量"安抚策略是否在起作用"的核心趋势指标。
连续负值（焦虑减少）→ 策略有效
连续正值（焦虑增加）→ 需要调整策略
剧烈波动 → 可能存在外部干扰因素
```

#### E07：月平静率

```
定义：一个月中有守护数据的天数中，完全没有焦虑事件的天数占比

为什么重要：
从长期角度衡量宠物整体情绪状态的改善。
月平静率持续上升 → 宠物逐渐适应独处
月平静率持续下降 → 可能需要行为训练或就医咨询
```

## 3. 数据采集方案

### 3.1 自动采集指标

以下指标由系统在守护过程中自动记录，无需用户干预：

| 数据项 | 采集方式 | 存储位置 |
|--------|---------|---------|
| 守护开始/结束时间 | 用户点击按钮触发 | guard_sessions |
| 焦虑事件触发时间 | 音频引擎自动检测 | anxiety_events |
| 峰值分贝 | RMS 计算 | anxiety_events.peak_db |
| 事件持续时长 | 从触发到冷却开始的时长 | anxiety_events.duration_sec |
| 播放的语音 ID | 随机选择时记录 | anxiety_events.voice_played_id |

### 3.2 用户反馈采集（后续版本）

| 指标 | 采集方式 | 用途 |
|------|---------|------|
| 安抚有效性（主人评估） | 报告页底部 赞/踩 按钮 | 验证分贝触发是否准确 |
| 回家后观察到异常行为 | 报告页备注输入框 | 补充分贝数据无法反映的行为（如破坏家具） |
| 邻居投诉 | 手动标记 | 严重事件追踪 |

## 4. AI 情绪总结引擎

### 4.1 Prompt 设计

```markdown
[System]
你是一位温柔的宠物情绪分析师，名字叫"宠博士"。
你需要根据提供的宠物守护数据，生成一段亲切温暖的情绪日报总结。

[Constraints]
- 字数控制在 50 字左右
- 语气像在跟老朋友聊天，亲切但专业
- 如果焦虑次数为 0，要夸奖宠物
- 如果有焦虑事件，先共情，再给正向引导
- 提到对比昨日的趋势
- 直接输出总结文字，不要任何前缀标记

[Input Template]
- 宠物名字：{petName}
- 守护时长：{durationMinutes} 分钟
- 焦虑事件次数：{anxietyCount} 次
- 峰值分贝：{peakDb} dB
- 峰值时间：{peakTime}
- 安抚成功率：{sootheRate}%
- 对比昨日：{vsYesterday}

[Examples]

案例 1（无焦虑）:
"今天旺财超级乖 🥰 整个上午安安静静的，完全是个省心的小天使！比昨天还安静，进步大大的～"

案例 2（有焦虑）:
"旺财今天下午 2 点左右闹了两次小脾气 🥺 可能是快递小哥路过的声音刺激到了。好在 AI 已经成功安抚啦，整体比昨天安静了 15%，宝宝在进步！"

案例 3（首次使用）:
"今天是旺财第一次被守护 👋 监测到 3 次焦虑波动，峰值在早上 9 点多，可能是你刚出门不久。AI 都及时介入了，之后有了对比数据就能看到趋势啦～"
```

### 4.2 降级模板（DeepSeek 不可用时）

```javascript
function generateFallbackSummary(stats) {
  if (stats.anxietyCount === 0) {
    return `${stats.petName}今天超级乖 🥰 全程安安静静，是个省心的小天使！`
  }
  const trend = stats.vsYesterday.includes('下降') ? '继续加油！' :
                stats.vsYesterday.includes('上升') ? '明天会更好～' :
                '我们一起关注～'
  return `今日${stats.petName}有些小情绪 🥺 监测到 ${stats.anxietyCount} 次焦虑波动，峰值在 ${stats.peakTime}。AI 已介入安抚，${trend}`
}
```

## 6. 评估体系局限性

### 6.1 诚实声明

本评估体系有以下已知局限：

| 局限 | 说明 | 影响 |
|------|------|------|
| 仅基于分贝 | 无法区分宠物叫声 vs 其他高分贝声音（摔门、警笛） | 可能高估焦虑事件数 |
| 无法检测无声焦虑 | 宠物在焦虑时也可能表现为静默（躲藏、发抖、破坏） | 可能低估真实焦虑程度 |
| 安抚效果无直接验证 | 播放语音后宠物是否安静下来，只能间接推断 | 安抚成功率基于"冷却期间无二次触发"的假设 |
| 采样偏差 | 主人只在特定时段开启守护 | 不能代表全天情绪状态 |

### 6.2 用户须知建议

> "本报告基于音频监测数据生成。由于环境噪音、麦克风灵敏度等因素，数据可能不完全反映宠物真实状态。如有严重行为问题，建议同时咨询兽医行为专家。"
